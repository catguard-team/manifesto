# OPERATIONS — 자경단 운영 규칙

> 자경단이 일하는 방식입니다. 강령은 사상이고, 이 문서는 절차에 해당합니다.
> 절차는 자주 바뀌고, 사상은 잘 안 바뀝니다.

---

## 1. 의사결정

### 1-1. 일상 작업

- 본인 판단으로 진행하셔도 됩니다.
- 의심스러우시면 디스코드 #그루밍 채널에 한 줄 던지시고 진행해 주세요.
- 사후 보고로 충분합니다. 사전 허락은 자경단의 결과 어울리지 않습니다.

### 1-2. 자경단 전체에 영향 있는 결정

다음은 디스코드 토론 + 자경단원 의견 수렴 후 단장이 결정합니다.

- 강령(`GANGNYANG.md`) 본문 변경
- 매니페스토(`MANIFESTO.md`) 본문 변경
- 신규 레포 생성 / 기존 레포 폐기
- 외부 단체와의 공식 협력
- 자경단 명의의 공개 입장 발표

### 1-3. 단장 단독 결정

다음은 단장이 단독으로 결정합니다.

- `SAFETY.md` 강화
- 자경단원 자격 박탈 (강령 명백 위반 시)
- GitHub org admin 권한 부여·회수
- 자금·후원 관련 모든 결정 (받을지 말지 포함)

### 1-4. 단장이 결정 못 하는 것

- `SAFETY.md` 완화 — 절대 불가합니다.
- 자경단의 영리화 — 자경단원 만장일치가 필요합니다.
- 자경단의 정치적 입장 표명 — 안 하는 게 기본값입니다.

## 2. 합류와 이탈

### 2-1. 합류

- 디스코드 합류 → 본인 자기소개 한 줄 → 그게 끝입니다.
- 별도 가입 절차나 심사, 가입비는 없습니다.
- "자경단원입니다"라고 본인이 선언하시면 자경단원이 됩니다.

### 2-2. 활동 의무

- **없습니다.**
- 한 달에 한 번도 활동하지 않으셔도 자경단원이십니다.
- 디스코드 ROM(Read Only Member)도 자경단원이십니다.

### 2-3. 이탈

- 디스코드를 나가시면 이탈입니다. 별도 절차는 없습니다.
- 인사하고 가셔도 되고, 그냥 가셔도 됩니다.
- 자경단 GitHub org에서 본인을 제거하고 싶으시면 단장에게 한 줄 보내주세요. 즉시 처리해 드립니다.
- 본인이 작성하신 PR·문서·코드는 자경단에 남습니다 (저작권 정리는 LICENSE 파일을 참조해 주세요).

### 2-4. 재합류

- 언제든 환영합니다.
- "왜 갔다 오셨어요?"는 자경단의 결과 어울리지 않는 질문입니다.

---

## 3. 작업 흐름

### 3-1. 작업 단위

- 작은 단위로 자주 합니다. 큰 변경은 잘게 쪼개주세요.
- 1 PR = 1 변경 원칙입니다. 검토하기 쉬워야 한다고 봅니다.

### 3-2. 브랜치 전략

- 단순화를 우선합니다 — 작은 수정은 `main` 직접 푸시도 OK입니다.
- 큰 변경이나 의견이 필요한 변경은 PR로 부탁드립니다.
- `feature/`, `fix/` 같은 prefix는 자유입니다. 강제하지 않습니다.

### 3-3. 커밋 메시지

- 한국어로 쓰셔도 됩니다.
- 자경단 톤을 유지해 주세요. 예: "꿀잠 스팟에 VS Code 한국어 팁 추가", "강령 7조 오타 수정".
- "Update README" 같은 의미가 잘 와닿지 않는 메시지는 피해주세요.
- Conventional Commits(`feat:`, `fix:`)는 강제하지 않습니다. 본인이 좋으시면 쓰셔도 됩니다.

### 3-4. 코드 리뷰 = 그루밍

- PR에 코멘트를 다실 때:
  - ✅ "이 부분 같이 봐도 될까요?"
  - ✅ "여기 X 방식도 있어요"
  - 자경단의 결과 어울리지 않는 표현: "이거 왜 이렇게 짰어요?", "이 코드는 별로네요"
- 비난은 자경단 톤이 아닙니다. 개선 제안은 환영합니다.
- 새끼고양이의 PR은 1차 머지를 우선하고, 개선은 후속 PR로 진행합니다.

GitHub에 잡담 이슈를 올리시거나, Discord에 코드 토론을 하시면 검색이 어려워집니다. 가능하면 채널 분리를 부탁드립니다.

## 6. 후원·재정

### 6-1. 기본 자세

- 자경단은 활동에 필요한 비용을 자경단원이 자비로 충당하는 것을 기본값으로 합니다.
- 다만 자비만으로 굴러가는 자경단은 지속가능하지 않다고 봅니다.
- 후원·정부 공모·기업 협력은 받습니다. 자경단의 지속가능성을 위해서입니다.
- 받되 활동의 동력·중심에 두지는 않으려고 합니다. 후원이 끊겨도 자경단은 굴러가야 한다고 생각합니다.

### 6-2. 후원 받을 때

- 모든 후원은 단장 1인 책임 하에 별도로 운영합니다.
- 후원 내역은 분기마다 디스코드에 한 줄로 공개합니다 ("○분기 후원 N원 / 지출 N원 / 내역 요약").
- 개별 후원자의 신원·금액은 비공개입니다 (개인정보).
- 자경단은 후원자에게 어떤 우대도 하지 않습니다.

### 6-3. 받지 않는 것

- 정치 자금
- 종교 단체 자금 (개인 후원자가 종교인이신 것은 무관합니다)
- 조건부 후원 ("자경단이 X를 해야 줌")
- 자경단의 데이터·접근권을 요구하는 후원

## 7. 외부 협력

### 7-1. 협력 가능

- 정부 공모 사업: 단발성 협력은 가능합니다. 자경단을 사업단체로 등록하지는 않습니다.
- 대기업 ESG: 자원·기기 기부는 받습니다. 자경단을 ESG 홍보 소재로 쓰시는 건 거절합니다.
- NGO·사회적기업: 새끼고양이 연계는 가능합니다. 자경단의 운영권 공유는 거절합니다.
- 학계·연구: 자경단 활동 연구는 가능합니다. 다만 `SAFETY.md`가 우선합니다.

### 7-2. 협력 결정 절차

- 디스코드 토론 → 자경단원 의견 → 단장 결정 순서로 진행합니다.
- 협력 사실은 [`CHANGELOG.md`](./CHANGELOG.md) 또는 별도 로그에 공개합니다.

## 8. 분쟁 해결

### 8-1. 자경단원 사이

- 1차: 당사자끼리 그루밍을 시도해 주세요.
- 2차: 디스코드 #그루밍에서 공개 토론.
- 3차: 단장 중재.
- 4차: 한 분 또는 양쪽이 자경단을 떠나시는 것까지 열어둡니다. 강요는 없습니다.

### 8-2. 단장과의 분쟁

- 단장도 자경단원 한 마리입니다. 동등하게 토론해 주세요.
- 단장이 잘못한 게 명확하면 단장이 사과하고 `CHANGELOG.md`에 기록합니다.
- 단장 교체 절차는 별도로 마련하지 않았습니다. 자경단이 거기까지 가게 되면, 자경단을 해체하는 게 더 자경단스러운 모습일 거라고 생각합니다.

## 9. 자경단의 끝

- 자경단의 일이 끝나면 자경단도 끝낸다는 입장입니다 (강령 10조).
- 잠수 자경단원을 추적하지 않습니다. 활동이 적은 시기는 자경단의 휴식이지 결함이 아니라고 봅니다.
- 해체는 자동 타이머로 정하지 않습니다. 단장이 "이제 자경단의 일이 끝났다"고 판단하면 디스코드에 한 줄 던지고, 30일 동안 자경단원의 명백한 반대가 없으면 해체를 진행합니다.
- 해체 시 GitHub 레포는 archive(읽기 전용 보존)합니다. 디스코드 서버는 90일 후 폐쇄합니다.
- 자경단의 영속성은 목표가 아닙니다. 자경단의 부재도 결함이 아니라고 봅니다.

## 11. 자료 공유 절차 (링크 등재)

자경단원이 자기 작업물(레포, 강의, 글, 도구)을 자경단 골목에 올리고 싶으실 때 따르는 절차입니다.

> **자경단의 역할 한계**: 자경단은 **자체 레포 외에는 사본을 두지 않습니다**. 다른 사람 레포는 **모으는 역할(링크 등재)**만 수행합니다. 미러·동기화·복제·아카이빙은 하지 않습니다.

### 11-1. 권리 모델

다른 사람 레포는 자경단 골목에 **사본이 없습니다**. 따라서:

- **소유권·유지보수권·접근권**: 처음부터 끝까지 **원작자**의 것입니다.
- **자경단의 역할**: 위험 검증 후 인덱스에 링크를 등재하고 골목에 공지하는 정도입니다.
- **보존**: 자경단이 책임지지 않습니다. 원본이 사라지면 링크도 깨집니다 (원작자 재량).
- **검증권**: 단장이 위험 7가지만 봅니다. 톤·문체·디자인은 검증하지 않습니다.

### 11-2. 절차

1. **자가 점검** — 원작자가 본인 자료를 위험 7가지(어린이 안전 / 노인 사기 / 자해 조장 / 차별·혐오 / 민감정보 / 저작권 / 명예훼손)로 한 번 봐주세요.
2. **공유 요청 issue** — `catguard-team/.github` 레포에 `share-request` 템플릿으로 issue를 작성해 주세요. 처음이시면 빈칸 그대로 채우시면 됩니다. 모르는 항목은 비워두고 보내셔도 괜찮습니다.
3. **단장 검증** — 단장이 위험 7가지만 점검합니다 (24~72h). 거절 시 사유를 공개하고, 수정 후 재요청도 가능합니다.
4. **링크 등재** — 단장이 `.github/profile/README.md` "자경단원 강의·자료" 섹션 또는 인덱스 레포(`kkulzam-spot`, `jwidori` 등)에 링크를 추가합니다.
5. **골목 공개** — 디스코드에 공지하고, 원작자분 동의 시 원본 레포에 자경단 검증 배지를 부착합니다.

### 11-3. 거절 사유 공개

- 검증 거절 시 사유를 issue에 명시합니다.
- 원작자가 위험을 수정하셔서 재요청하시면 다시 검증합니다.
- "톤이 별로다", "디자인이 안 예쁘다"는 거절 사유가 될 수 없습니다 (3대 존중 — 개인 존중).

### 11-4. 이의 제기

- 단장의 검증 결과에 동의하지 않으시면 디스코드 #그루밍 채널에서 공개 토론 가능합니다.
- 자경단원 과반 동의 시 단장 결정을 번복할 수 있습니다.

### 11-5. 원작자가 자경단을 떠나실 때

- 링크 등재는 그대로 두실 수 있습니다 (자경단원이 떠나셔도 자료는 자료입니다).
- 원작자가 명시적으로 빼달라 요청하시면 즉시 빼드립니다 (사본이 없으니 유예 기간이 별도로 필요하지 않습니다).

### 11-6. 링크 빼기 요청

- `catguard-team/.github`에 `remove-request` 템플릿으로 issue를 보내주세요.
- **즉시 빼드립니다** — 사본이 없으니 1주일 유예가 의미를 갖기 어렵습니다. 단장이 issue 받으면 24h 안에 인덱스에서 제거합니다.
- 다른 PR/이슈/문서 안에 인용된 부분은 남을 수 있습니다 (저작권은 원작자분 것입니다).

## 12. 지속가능성

자경단이 오래 굴러가려면 자원이 들어와야 한다고 생각합니다. 자경단원의 시간, 자비 충당, 후원, 정부 공모, 기업 협력, 자경단원의 유료 작업물 — 이 모두가 지속가능성에 기여합니다.

저희는 무조건 무료 자선 단체가 아닙니다. 무료가 더 자연스러운 자리도 있고, 유료가 더 자연스러운 자리도 있다고 봅니다.

### 12-1. 무료가 자연스러운 영역

자경단이 직접 만들고 운영하는 자산은 무료로 공개합니다.

- 매니페스토·강령·운영 규칙 (자경단의 정체성)
- 새끼고양이 학습 핸드북 (어린이·노묘·자영업자·접근성 트랙)
- 꿀잠 스팟 (자경단원이 모은 팁)
- 위험 검증 결과 (안전 정책)

### 12-2. 유료가 자연스러운 영역

자경단원 개인이 본인 영역에서 만드시는 작업물 중 다음은 유료가 자연스럽다고 봅니다.

- 자경단원이 만든 API·서비스 (서버·트래픽 비용이 드는 인프라)
- 깊이 있는 유료 강의 (자경단원이 시간을 들여 만든 큰 작업)
- 유료 도구·라이브러리·SaaS (지속 개발이 필요한 것)
- 유료 컨설팅·1:1 코칭

이 모두 사회 환원의 한 형태라고 봅니다. 무료 오픈소스만이 환원의 유일한 길은 아니라고 생각해요. 잘 만들어진 유료 도구가 누군가의 시간을 아껴드리고, 운영비를 감당하면서 지속될 때, 그 자체가 사회에 가치를 더하는 일이라고 봅니다.

### 12-3. 인덱스 등재 시 (§11 보완)

[§11](#11-자료-공유-절차-링크-등재) 절차에 따라 자경단원의 작업물을 인덱스에 링크 등재할 때, **유료 여부는 거절 사유가 되지 않습니다**. 단장은 위험 7가지(어린이 안전 / 노인 사기 / 자해 조장 / 차별·혐오 / 민감정보 / 저작권 / 명예훼손)만 점검합니다. 가격이 있는지 없는지는 보지 않습니다.

다만 자경단원이 자료를 등재하실 때 **유료 표시는 명확히 해주시면 좋습니다** — 인덱스에 "유료" 라벨을 부착합니다. 새끼고양이가 클릭하시기 전에 알 수 있도록.

### 12-4. 자경단 자체의 수익 사업

자경단은 (org 차원에서는) 수익 사업을 하지 않습니다. 자경단원 개인이 본인 영역에서 수익 활동을 하시는 것은 자유이고, 자경단의 인덱스에 그 작업물을 등재하실 수 있습니다 ([§11](#11-자료-공유-절차-링크-등재) + 위 §12-3).

### 12-5. 지속가능성과 영리화 구분

[`GANGNYANG.md` 제1조](./GANGNYANG.md) "외부 권위에 의존하지 않는다"는 그대로 유지됩니다. 자경단의 의사결정은 자경단원이 합니다. 자경단원이 본인 작업물로 수익을 내시는 것과 자경단 자체가 영리 단체로 변하는 것은 다른 문제입니다 ([§1-4](#1-4-단장이-결정-못-하는-것) — "자경단의 영리화"는 자경단원 만장일치가 필요합니다).
