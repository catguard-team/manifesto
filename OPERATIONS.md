# OPERATIONS — 자경단 운영 규칙

> 자경단이 일하는 방식. 강령은 사상이고, 이 문서는 절차다.
> 절차는 자주 바뀐다. 사상은 안 바뀐다.

---

## 1. 의사결정

### 1-1. 일상 작업

- 본인 판단으로 진행 가능.
- 의심스러우면 디스코드 #그루밍 채널에 한 줄 던지고 한다.
- 사후 보고로 충분하다. 사전 허락은 자경단 문화가 아니다.

### 1-2. 자경단 전체에 영향 있는 결정

다음은 디스코드 토론 + 자경단원 의견 수렴 후 단장이 결정한다.

- 강령(`GANGNYANG.md`) 본문 변경
- 매니페스토(`MANIFESTO.md`) 본문 변경
- 신규 레포 생성 / 기존 레포 폐기
- 외부 단체와의 공식 협력
- 자경단 명의의 공개 입장 발표

### 1-3. 단장 단독 결정

- `SAFETY.md` 강화
- 자경단원 자격 박탈 (강령 명백 위반 시)
- GitHub org admin 권한 부여/회수
- 자금/후원 관련 모든 결정 (받을지 말지 포함)

### 1-4. 단장이 결정 못 하는 것

- `SAFETY.md` 완화 — 절대 불가
- 자경단의 영리화 — 자경단원 만장일치 필요
- 자경단의 정치적 입장 표명 — 안 함이 기본값

---

## 2. 합류와 이탈

### 2-1. 합류

- 디스코드 합류 → 본인 자기소개 한 줄 → 끝.
- 별도 가입 절차, 심사, 가입비 없음.
- "자경단원입니다"라고 본인이 선언하면 자경단원이다.

### 2-2. 활동 의무

- **없다.**
- 한 달에 한 번도 활동 안 해도 자경단원이다.
- 디스코드 ROM(Read Only Member)도 자경단원이다.

### 2-3. 이탈

- 디스코드 나가기 = 이탈. 별도 절차 없음.
- 인사하고 가도 되고, 그냥 가도 된다.
- 자경단 GitHub org에서 본인 제거하고 싶으면 단장에게 한 줄. 즉시 처리.
- 본인이 작성한 PR/문서/코드는 자경단에 남는다(저작권 정리는 LICENSE 파일 참조).

### 2-4. 재합류

- 언제든 환영.
- "왜 갔다 왔어요?" 묻지 않는다.

## 3. 작업 흐름

### 3-1. 작업 단위

- 작은 단위로 자주. 큰 변경은 잘게 쪼갠다.
- 1 PR = 1 변경 원칙. 검토하기 쉬워야 한다.

### 3-2. 브랜치 전략

- 단순화: `main` 직접 푸시 OK (작은 수정).
- 큰 변경, 의견 필요한 변경은 PR.
- `feature/`, `fix/` 같은 prefix는 자유. 강제 안 함.

### 3-3. 커밋 메시지

- 한국어 OK.
- 자경단 톤 유지. 예: "꿀잠 스팟에 VS Code 한국어 팁 추가", "강령 7조 오타 수정".
- "Update README" 같은 무의미 메시지 지양.
- Conventional Commits(`feat:`, `fix:`) 강제 안 함. 본인이 좋으면 써도 됨.

### 3-4. 코드 리뷰 = 그루밍

- PR에 코멘트 달 때:
  - ✅ "이 부분 같이 봐도 될까요?"
  - ✅ "여기 X 방식도 있어요"
  - ❌ "이거 왜 이렇게 짰어요?"
  - ❌ "이 코드는 별로네요"
- 비난 금지. 개선 제안 환영.
- 새끼고양이 PR은 1차 머지 우선, 개선은 후속 PR.

## 4. 채널 분리

| 용도 | 어디서 |
|------|-------|
| 코드, 문서, 이슈, PR | GitHub |
| 모집, 공지, 잡담 | Discord |
| 멘토링 1:1 | Discord 공개방(미성년자) / DM(성인만) |
| 운영 결정 | Discord #그루밍 또는 GitHub Discussions |
| 후원·재정 | 비공개 (별도 운영) |

GitHub에 잡담 이슈, Discord에 코드 토론은 가능하면 피한다. 검색 안 됨.

## 5. 디스코드 채널 구조

```
📢 공지방
🐾 잡담방
🤝 그루밍방        ← 운영 토론, 강령 토의
🆘 도와줘방        ← 막힌 거 물어보기 (자경단원·새끼고양이 공용)
🎯 사냥일지방      ← 활동 인증 (선택)
🚨 비상방          ← SAFETY 사건 대응 (제한 권한)
```

채널 신설/통합은 단장 결정. 의견은 #그루밍에서.

## 6. 후원·재정

### 6-1. 기본 자세

- 자경단은 활동에 필요한 비용을 자경단원이 자비로 충당하는 것을 기본값으로 한다.
- 단, 자비만으로 굴러가는 자경단은 지속가능하지 않다는 걸 안다.
- 후원·정부 공모·기업 협력은 받는다. 자경단의 지속가능성을 위해.
- 받되 활동의 동력·중심에 두지 않는다. 후원이 끊겨도 자경단은 굴러가야 한다.

### 6-2. 후원 받을 때

- 모든 후원은 단장 1인 책임 하에 별도 운영.
- 후원 내역은 분기마다 디스코드에 한 줄 공개("○분기 후원 N원 / 지출 N원 / 내역 요약").
- 개별 후원자 신원·금액은 비공개(개인정보).
- 자경단은 후원자에게 어떤 우대도 하지 않는다.

### 6-3. 받지 않는 것

- 정치 자금
- 종교 단체 자금 (개인 후원자가 종교인인 것은 무관)
- 조건부 후원 ("자경단이 X를 해야 줌")
- 자경단의 데이터·접근권을 요구하는 후원

## 7. 외부 협력

### 7-1. 협력 가능

- 정부 공모 사업: 단발성 협력은 OK. 자경단을 사업단체로 등록하지 않음.
- 대기업 ESG: 자원·기기 기부는 OK. 자경단을 ESG 홍보 소재로 쓰는 건 거부.
- NGO·사회적기업: 새끼고양이 연계는 OK. 자경단의 운영권 공유는 거부.
- 학계·연구: 자경단 활동 연구는 OK. 단, `SAFETY.md` 우선.

### 7-2. 협력 결정 절차

- 디스코드 토론 → 자경단원 의견 → 단장 결정.
- 협력 사실은 [`CHANGELOG.md`](./CHANGELOG.md) 또는 별도 로그에 공개.

---

## 8. 분쟁 해결

### 8-1. 자경단원 간

- 1차: 당사자끼리 그루밍 시도.
- 2차: 디스코드 #그루밍에서 공개 토론.
- 3차: 단장 중재.
- 4차: 한 명 또는 양쪽이 자경단을 떠난다. 강요는 없다.

### 8-2. 단장과의 분쟁

- 단장도 자경단원 한 마리다. 동등하게 토론.
- 단장이 잘못한 게 명확하면 단장이 사과 + `CHANGELOG.md`에 기록.
- 단장 교체 절차는 별도 마련 안 함. 자경단이 거기까지 가면 자경단을 해체하는 게 더 자경단스럽다.

---

## 9. 자경단의 끝

- 자경단의 일이 끝나면 자경단도 끝낸다(강령 10조).
- 잠수 자경단원을 추적하지 않는다. 활동이 적은 시기는 자경단의 휴식이지 결함이 아니다.
- 해체는 자동 타이머로 정하지 않는다. 단장이 "이제 자경단의 일이 끝났다"고 판단하면 디스코드에 한 줄 던지고, 30일 동안 자경단원의 명백한 반대가 없으면 해체 진행.
- 해체 시 GitHub 레포는 archive(읽기 전용 보존). 디스코드 서버는 90일 후 폐쇄.
- 자경단의 영속성은 목표가 아니다. 자경단의 부재도 결함이 아니다.

- 변경 이력은 [`CHANGELOG.md`](./CHANGELOG.md)에 한 줄.

## 11. 자료 공유 절차 (링크 등재)

자경단원이 자기 작업물(레포, 강의, 글, 도구)을 자경단 골목에 올리고 싶을 때 따르는 절차.

> **자경단의 역할 한계**: 자경단은 **자체 레포 외에는 사본을 두지 않는다**. 다른 사람 레포는 **모으는 역할(링크 등재)**만 수행한다. 미러·동기화·복제·아카이빙 안 한다.

### 11-1. 권리 모델

다른 사람 레포는 자경단 골목에 **사본이 없다**. 따라서:

- **소유권·유지보수권·접근권**: 처음부터 끝까지 **원작자**.
- **자경단의 역할**: 위험 검증 후 인덱스에 링크 등재 + 골목 공지.
- **보존**: 자경단이 책임지지 않는다. 원본이 사라지면 링크도 깨진다 (원작자 재량).
- **검증권**: 단장 — 위험 7가지만 본다. 톤·문체·디자인은 검증하지 않음.

### 11-2. 절차

1. **자가 점검** — 원작자가 본인 자료를 위험 7가지(어린이 안전 / 노인 사기 / 자해 조장 / 차별·혐오 / 민감정보 / 저작권 / 명예훼손)로 한 번 본다.
2. **공유 요청 issue** — `catguard-team/.github` 레포에 `share-request` 템플릿으로 issue 작성. 처음이면 빈칸 그대로 채우면 된다. 모르는 항목은 비워두고 보내도 OK.
3. **단장 검증** — 단장이 위험 7가지만 점검(24~72h). 거절 시 사유 공개 + 수정 후 재요청 가능.
4. **링크 등재** — 단장이 `.github/profile/README.md` "자경단원 강의·자료" 섹션 또는 인덱스 레포(`kkulzam-spot`, `jwidori` 등)에 링크 추가.
5. **골목 공개** — 디스코드 공지, 원본 레포에 자경단 검증 배지 부착(원작자 동의 시).

### 11-3. 거절 사유 공개

- 검증 거절 시 사유를 issue에 명시.
- 원작자가 위험을 수정해서 재요청하면 다시 검증.
- "톤이 별로다", "디자인이 안 예쁘다"는 거절 사유가 될 수 없다(3대 존중 — 개인 존중).

### 11-4. 이의 제기

- 단장의 검증 결과에 동의하지 않으면 디스코드 #그루밍 채널에서 공개 토론.
- 자경단원 과반 동의 시 단장 결정을 번복 가능.

### 11-5. 원작자가 자경단을 떠날 때

- 링크 등재는 그대로 둘 수 있다 (자경단원이 떠나도 자료는 자료).
- 원작자가 명시적으로 빼달라 요청하면 즉시 빼기 (사본이 없으니 유예 기간 불필요).

### 11-6. 링크 빼기 요청

- `catguard-team/.github`에 `remove-request` 템플릿으로 issue.
- **즉시 빼기** — 사본이 없으니 1주일 유예가 의미 없다. 단장이 issue 받으면 24h 안에 인덱스에서 제거.
- 다른 PR/이슈/문서 안에 인용된 부분은 남을 수 있다(저작권은 원작자 것).

## 12. 지속가능성

자경단이 오래 굴러가려면 자원이 들어와야 한다고 생각합니다. 자경단원의 시간, 자비 충당, 후원, 정부 공모, 기업 협력, 자경단원의 유료 작업물 — 이 모두가 지속가능성에 기여합니다.

저희는 무조건 무료 자선 단체가 아닙니다. 무료가 더 자연스러운 자리도 있고, 유료가 더 자연스러운 자리도 있다고 봅니다.

### 12-1. 무료가 자연스러운 영역

자경단이 직접 만들고 운영하는 자산은 무료로 공개합니다.

- 매니페스토·강령·운영 규칙 (자경단의 정체성)
- 새끼고양이 학습 핸드북 (어린이·노묘·자영업자·접근성 트랙)
- 꿀잠 스팟 (자경단원이 모은 팁)
- 위험 검증 결과 (안전 정책)

### 12-2. 유료가 자연스러운 영역

자경단원 개인이 본인 영역에서 만드시는 작업물 중 다음은 유료가 자연스럽다고 봅니다.

- 자경단원이 만든 API·서비스 (서버·트래픽 비용이 드는 인프라)
- 깊이 있는 유료 강의 (자경단원이 시간을 들여 만든 큰 작업)
- 유료 도구·라이브러리·SaaS (지속 개발이 필요한 것)
- 유료 컨설팅·1:1 코칭

이 모두 사회 환원의 한 형태라고 봅니다. 무료 오픈소스만이 환원의 유일한 길은 아니라고 생각해요. 잘 만들어진 유료 도구가 누군가의 시간을 아껴드리고, 운영비를 감당하면서 지속될 때, 그 자체가 사회에 가치를 더하는 일이라고 봅니다.

### 12-3. 인덱스 등재 시 (§11 보완)

[§11](#11-자료-공유-절차-링크-등재) 절차에 따라 자경단원의 작업물을 인덱스에 링크 등재할 때, **유료 여부는 거절 사유가 되지 않습니다**. 단장은 위험 7가지(어린이 안전 / 노인 사기 / 자해 조장 / 차별·혐오 / 민감정보 / 저작권 / 명예훼손)만 점검합니다. 가격이 있는지 없는지는 보지 않습니다.

다만 자경단원이 자료를 등재하실 때 **유료 표시는 명확히 해주시면 좋습니다** — 인덱스에 "유료" 라벨을 부착합니다. 새끼고양이가 클릭하기 전에 알 수 있도록.

### 12-4. 자경단 자체의 수익 사업

자경단은 (org 차원에서는) 수익 사업을 하지 않습니다. 자경단원 개인이 본인 영역에서 수익 활동을 하시는 것은 자유이고, 자경단의 인덱스에 그 작업물을 등재하실 수 있습니다 ([§11](#11-자료-공유-절차-링크-등재) + 위 §12-3).

### 12-5. 지속가능성과 영리화 구분

[`GANGNYANG.md` 제1조](./GANGNYANG.md) "외부 권위에 의존하지 않는다"는 그대로 유지됩니다. 자경단의 의사결정은 자경단원이 합니다. 자경단원이 본인 작업물로 수익을 내시는 것과 자경단 자체가 영리 단체로 변하는 것은 다른 문제입니다 ([§1-4](#1-4-단장이-결정-못-하는-것) — "자경단의 영리화"는 자경단원 만장일치 필요).
